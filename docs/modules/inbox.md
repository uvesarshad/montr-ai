# Omnichannel Inbox Module

> Scope: Unified messaging hub — channels, conversations, AI assist, CRM integration.
> Rendering context: Client-side with Socket.io real-time
> Project tier: 4
> Last updated: 2026-06-04

## Overview

The Omnichannel Inbox consolidates messages from multiple channels (WhatsApp, Email, Webchat, Telegram, Instagram, Facebook, SMS, and others) into a unified conversation view. It includes AI-assisted reply generation, CSAT automation, chatbot builder integration, knowledge base linking, conversation assignment, and CRM contact linking.

## Entry Points

- src/app/(app)/inbox/ — Inbox pages (conversation list/thread, channel setup, analytics, leaderboard, knowledge base, chatbots).
- src/app/api/v2/inbox/ — Inbox API routes (channels, conversations, messages, assignments, labels).
- src/app/api/v2/inbox/webhook/ — Inbound webhook receivers for each channel provider (signature-verified, public).

## Key Pages

src/app/(app)/inbox/page.tsx — Conversation list and message thread with filters (channel, status, assignee) and reply compose.

## Data Models

inbox_channels — Channel configurations. Fields: organizationId, type (whatsapp/email/webchat/telegram/instagram/facebook/sms/...), name, credentials (encrypted), status, settings.

inbox_conversations — Conversation threads. Fields: organizationId, channelId, channelType, contactId, assigneeId, status (open/resolved/pending), labels, lastMessageAt, metadata (platform-specific IDs).

inbox_messages — Individual messages within a conversation. Fields: conversationId, role (incoming/outgoing), content, mediaUrls, sentAt, status (sent/delivered/read/failed), platformMessageId.

inbox_members — Team member access to inbox (which agents can see/handle which channels). Fields: organizationId, userId, channelIds, role.

inbox_labels — Conversation tags/categories. Fields: organizationId, name, color.

knowledge_base — AI knowledge entries used for reply suggestions and semantic search (src/lib/db/models/knowledge-base.model.ts). Fields: organizationId, brandId (brand scoping, ObjectId), name, description, type (document/url/text/faq/pdf/social_report/crm_contact/crm_deal/inbox_thread/form/ai_memory), sourceModule (manual/copilot/crm/social/inbox/forms/documents), content, metadata (title/url/fileName/fileSize/mimeType/author/tags), embedding (number array, not selected by default), embeddingModel, chunks (array of {content, embedding, startIndex, endIndex}), isActive, createdById.

## Real-Time Updates

use-inbox-socket.ts connects to Socket.io at /api/socket and joins conversation rooms. Incoming messages are pushed via the channel webhook handler → inbox service → Socket.io emit to the conversation room. The conversation list and message thread update in real-time without polling.

## Chatbot Integration

The inbox supports embedding AI chatbots via the chatbot builder (src/app/(app)/ai-bots/). Chatbot configurations include system prompt, knowledge base, and response style. Chatbot replies are handled by src/lib/inbox/chatbot-ai-reply.ts and rate-limited by src/lib/inbox/chatbot-rate-limiter.ts.

Chatbot embed: src/lib/inbox/chatbot-widget-script.ts generates the embeddable JS snippet for website integration.

AGENT NOTE: Chatbot origin validation (src/lib/inbox/chatbot-origin.ts) checks the Referer against allowed domains to prevent unauthorized chatbot usage.

## AI-Assisted Reply

AI reply suggestions are generated by src/lib/inbox/ai-suggestions.service.ts. It reads the conversation history and knowledge base context, then calls generateTextWithClient.

## CRM Integration

Conversations can be linked to CRM contacts. When a conversation is resolved, CSAT automation (src/lib/inbox/csat-automation.service.ts) sends a satisfaction survey.

## Channel Webhook Receivers

Each supported channel has an inbound webhook endpoint under /api/v2/inbox/webhook/[channel]. Each verifies the provider's signature format (HMAC, challenge-response, etc.) before processing.
